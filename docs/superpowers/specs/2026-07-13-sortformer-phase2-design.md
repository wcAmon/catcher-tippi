# Sortformer Phase 2: Streaming, Fusion, and Traditional Chinese Output

- Date: 2026-07-13
- Status: approved for planning
- Parent design: `docs/plans/2026-07-12-sortformer-diarization-design.md`
  (phase 2 of 3; read its "As-built corrections" section first)
- Predecessor: Phase 1 (engine) merged at `350a980`; offline `catcher diarize`
  works, INT8 artifact published as `wcamon/catcher-diar-mlx-int8`.

## Goal

After Phase 2, `catcher transcribe --model <asr> --diar-model <diar>
--audio x.wav` prints who-said-what-when in Traditional Chinese, and the C
ABI v2 exposes speaker-segmented output so Phase 3 only has to build UI.

## Scope (six work blocks)

1. Rewrite scalar attention loops as MLX matmuls (performance prerequisite).
2. `StreamingDiarizer` in `sortformer-mlx` (AOSC streaming, low-latency preset).
3. Token timestamps through `nemotron-mlx`'s `StreamingTranscriber`.
4. `fusion` module: tokens × speaker probabilities → speaker segments.
5. `opencc` module: s2twp conversion via `ferrous-opencc`.
6. C ABI v2 + `catcher transcribe --diar-model` CLI integration.

Out of scope: all Tippi/Swift UI work (Phase 3), except the one-line
`CatcherClient.swift` fix needed to keep the app compiling.

## Decisions already made

- **Streaming semantics follow NeMo's synchronous `forward_streaming` path**
  (the official evaluation path), not the pipecat real-time service. The
  pipecat path hardcodes left/right context to 1/1, which disagrees with the
  published low-latency preset (1/7); references generated by hooking
  `forward_streaming` validate our implementation cleanly.
- **Speaker-cache update period is 188.** NeMo's demo preset tries to set
  `spkcache_refresh_rate = 144`, but the installed NeMo 2.7.3 module has no
  such attribute; the effective value is `spkcache_update_period = 188`. Our
  references come from this venv, so we match its actual behavior.
- **`catcher_create` changes signature in place** (adds `diar_model_path`).
  No `_v2` suffix: the ABI has never shipped externally and the only consumer
  (Tippi) lives in this repo.
- **OpenCC via `ferrous-opencc` 0.4** (pure Rust, Apache-2.0, all standard
  dictionaries embedded, FST lookup). Config: **s2twp** (Taiwan standard +
  Taiwan phrases).
- **CLI shape**: extend `catcher transcribe` with `--diar-model`; no new
  subcommand.
- **Attention matmul rewrite is in scope** (user-approved): streaming re-runs
  a ~390-frame full forward ~2×/second; the Phase 1 scalar loops cannot keep
  up in real time.

## Block 1: attention matmul rewrite (`sortformer-mlx`)

Replace the O(T²) scalar attention loops in `src/model/encoder.rs` and
`src/model/transformer.rs` with MLX ops (QKᵀ matmul → scale/bias → softmax →
×V on GPU). Two pieces of standing debt are folded in:

- Parse `xscaling` from `config.json` instead of hardcoding `sqrt(512)`.
- Consolidate the Norm/relu/softmax helpers duplicated between `encoder.rs`
  and `transformer.rs` into a shared `ops` module.

**Acceptance: every existing Phase 1 parity test passes unchanged** (mel
+0.24% rms gate 2%; encoder +0.04% gate 5%; end-to-end probability gates
max-abs 0.05 / mean 0.01). This is a behavior-preserving rewrite; no fixture
regeneration.

## Block 2: `StreamingDiarizer` (`sortformer-mlx/src/stream.rs`)

Push-based streaming diarization implementing NeMo's synchronous AOSC path.

### Preset constants (v2.1 low latency)

| parameter | value | note |
|---|---|---|
| frame duration | 80 ms | 10 ms hop × 8× subsampling |
| chunk_len | 6 frames | 0.48 s |
| chunk_left_context | 1 frame | |
| chunk_right_context | 7 frames | look-ahead |
| fifo_len | 188 frames | |
| spkcache_len | 188 frames | |
| spkcache_update_period | 188 frames | effective venv value (see Decisions) |
| spkcache_sil_frames_per_spk | 3 | |
| pred_score_threshold | 0.25 | |
| scores_boost_latest | 0.05 | |
| sil_threshold | 0.2 | |
| strong_boost_rate | 0.75 | |
| weak_boost_rate | 1.5 | |
| min_pos_scores_rate | 0.5 | |

Algorithmic latency = (chunk_len + right_context) × 80 ms = **1.04 s**.
Checkpoint config values (`chunk_len 188`, `fifo_len 0`, rc 1 in
`tests/fixtures/sortformer_config.json`) are training defaults; the preset
above overrides them at construction.

### Public API

```rust
pub struct StreamingDiarizer { /* Diarizer parts + StreamingState + audio buffer */ }

impl StreamingDiarizer {
    pub fn new(model_dir: &Path) -> ModelResult<Self>;          // or from Diarizer parts
    /// Push arbitrary 16 kHz mono samples; returns per-frame probabilities
    /// for chunk frames finalized by this push (may be empty).
    pub fn push_samples(&mut self, audio: &[f32]) -> ModelResult<Vec<[f32; 4]>>;
    /// Flush: process remaining audio as a final (padded) chunk.
    pub fn finish(&mut self) -> ModelResult<Vec<[f32; 4]>>;
    pub fn reset(&mut self);
    pub fn frame_ms(&self) -> u64;                              // 80
}
```

### Per-step pipeline

1. Accumulate audio; once a chunk's worth of mel frames (chunk_len×8 = 48 mel
   frames, plus contexts) is available, compute mel incrementally with
   look-back. The centered STFT window means each mel frame depends only on
   ±(n_fft/2) samples, so incremental mel equals whole-signal mel **exactly**
   (bit-for-bit), not approximately. Mirror `AudioChunkScheduler`'s pattern
   from `nemotron-mlx/src/model/stream.rs`.
2. `pre_encode` (dw-striding conv subsampling ÷8) runs **only on the chunk's
   mel** `[left_ctx | chunk | right_ctx]` — NeMo has no conv state cache; the
   contexts exist to absorb conv boundary effects. Output: 512-dim embeddings.
3. Concatenate `[spkcache | fifo | chunk_embs]` and run the **full 17-layer
   encoder with pre_encode bypassed** (new `Encoder::forward_embedded`,
   NeMo's `bypass_pre_encode=True`), then `encoder_proj` (512→192), 18-layer
   Transformer, sigmoid head. Full bidirectional attention; no KV cache —
   recurrence lives entirely in the AOSC embedding state.
4. Keep only the chunk region's predictions (drop spkcache/fifo/context
   frames); append to output.
5. State update (NeMo `streaming_update`, sortformer_modules.py:526–609):
   - Push chunk embeddings + preds into FIFO.
   - On FIFO overflow, pop `spkcache_update_period` frames (clamped) into the
     speaker cache; update the running mean-silence embedding
     (`mean_sil_emb`, `n_sil_frames`) from popped frames whose summed preds
     < `sil_threshold`.
   - If spkcache exceeds `spkcache_len`, compress (AOSC,
     `_compress_spkcache`, :838–896): score frames by log-odds
     (`log p − log(1−p) + Σ log(1−p) − log 0.5`); disable non-speech and
     (when a speaker has ≥ `min_pos` positive frames) its non-positive
     frames; boost the latest frames by `scores_boost_latest`; per-speaker
     quota 44 speech frames (strong floor 33 via ×2 boost, weak cap 66 via
     ×1 boost); reserve 3 silence slots per speaker filled with
     `mean_sil_emb`; take global top-k then **sort selected indices to
     preserve arrival order**.

### Numeric validation

New `tools/generate_sortformer_streaming_reference.py` (same venv,
`/tmp/sortformer-venv`) hooks `SortformerEncLabelModel.forward_streaming` /
`forward_streaming_step` with the low-latency preset and captures, for a
fixture WAV over N chunks: per-chunk predictions and state snapshots
(spkcache, fifo, mean_sil_emb, n_sil_frames) at selected steps. Rust tests
assert per-chunk parity on the INT8 artifact with gates of the same order as
Phase 1's end-to-end gates. Key NeMo anchors:
`sortformer_diar_models.py` `forward_streaming` :627–716,
`forward_streaming_step` :718–815, `frontend_encoder` :277–304;
`sortformer_modules.py` `streaming_feat_loader` :208–256,
`streaming_update` :526–609, `_compress_spkcache` :838–896.
The fixture must be long enough to exercise at least one FIFO pop **and one
spkcache compression** (> 188 + 188 + margin frames ≈ 35 s).

Pure-Rust unit tests (no model) cover the AOSC compression rule on synthetic
inputs: per-speaker quotas, strong/weak boosts, silence-slot reservation,
arrival-order preservation, silence-profile accumulation.

## Block 3: token timestamps (`nemotron-mlx`)

```rust
pub struct TimedToken { pub id: u32, pub frame: u64 }  // frame × 80 ms = time
```

- `StreamingRnntDecoder::decode_frames` enumerates encoder frames and tags
  each emitted token with its local frame index
  (`rnnt.rs:365` — `chunks_exact(1024).enumerate()`).
- `StreamingTranscriber` keeps a global encoder-frame counter (advanced by
  `emitted_frames_per_chunk` per decoded window) and offsets local indices.
- `push_samples` / `finish` / `transcribe_samples` return `Vec<TimedToken>`
  instead of `Vec<u32>`. All in-repo call sites (CLI, FFI, tests) update in
  the same change; no deprecated shim is kept.

Validation: unit test that greedy decode assigns the correct frame to tokens
emitted across frame boundaries (synthetic joint scores); an equivalence test
that chunked and offline decoding produce identical `(id, frame)` sequences
on the reference WAV.

## Block 4: fusion (`nemotron-mlx/src/fusion.rs`)

A pure state machine on the shared 80 ms grid. It depends only on
`TimedToken`, `[f32; 4]` probability frames, and a
`&dyn Fn(&[u32]) -> String`-style detokenizer handle (tokenizer + s2twp
applied by the caller closure) — no dependency on `sortformer-mlx`.

```rust
pub struct SpeakerSegment {
    pub speaker: u8,
    pub start_ms: u64,
    pub end_ms: u64,
    pub text: String,
    pub is_final: bool,
}

pub struct Fusion { /* config + pending tokens + diar horizon + segments */ }

impl Fusion {
    pub fn new(config: FusionConfig) -> Self;   // threshold, ±window, min turn
    pub fn push_tokens(&mut self, tokens: &[TimedToken]);
    pub fn push_diar_frames(&mut self, frames: &[[f32; 4]]);
    /// Current segment list: finalized segments plus at most one tentative tail.
    pub fn segments(&self, detok: impl Fn(&[u32]) -> String) -> Vec<SpeakerSegment>;
    pub fn flush(&mut self);                    // stop: finalize everything
    pub fn reset(&mut self);
}
```

Rules (from the parent design):

1. Token attribution: smooth the 4 speaker probabilities over a ±3-frame
   window around the token's frame; attribute to the most active speaker
   above the activity threshold (0.5, matching `catcher diarize`'s default).
2. No active speaker (silence/ambiguous overlap) → inherit the previous
   token's speaker; the very first token with no active speaker → speaker 0.
3. Group consecutive same-speaker tokens into segments.
4. Anti-flicker: a finalized turn shorter than 500 ms merges into the
   neighboring segment (prefer the previous one).
5. Tentative tail: tokens newer than the diarization horizon (last frame
   with diar output) form one tentative segment attributed to the current
   speaker, `is_final = false`; re-evaluated as diar frames arrive.
6. `flush()` finalizes the tail with whatever diar frames exist.

Testing: synthetic probability grids and token timings only — attribution,
inheritance, anti-flicker merging, tentative→final transitions, flush,
reset. No model artifacts involved.

## Block 5: opencc (`nemotron-mlx/src/opencc.rs`)

Thin wrapper over `ferrous-opencc` with the embedded **s2twp** config,
initialized once (`OnceLock`). Applied at exactly two places:

- segment text production (inside the detokenizer closure handed to
  `Fusion::segments`), and
- `catcher_text` / CLI plain-transcript output.

Conversion is idempotent on Traditional/non-Chinese text, so the tentative
tail can be re-converted wholesale on each update.

Tests: fixture conversions (软件→軟體, 信息→資訊, 里面→裡面, 鼠标→滑鼠),
idempotency on already-Traditional text, pass-through on English.

## Block 6: C ABI v2 + CLI

### Header changes (`crates/catcher-ffi/include/catcher.h`)

```c
catcher_handle_t *catcher_create(const char *asr_model_path,
                                 const char *diar_model_path, /* NULL = ASR only */
                                 const char *language,
                                 uint32_t lookahead);
/* start / push_audio / finish / text / last_error / destroy unchanged */
const char *catcher_segments(const catcher_handle_t *h); /* UTF-8 JSON array */
const char *catcher_warning(const catcher_handle_t *h);  /* NULL = no warning */
```

- `catcher_segments` returns
  `[{"speaker":0,"start_ms":1040,"end_ms":5200,"text":"…","final":true},…]`
  — borrowed pointer owned by the handle, same lifetime convention as
  `catcher_text`. With `diar_model_path = NULL` it returns `[]`; a single
  segment is NOT synthesized — ASR-only callers keep using `catcher_text`.
- `catcher_push_audio` feeds the same samples to both engines (each buffers
  independently; their chunk cadences differ), then lets fusion consume both
  outputs. `catcher_text` now returns the s2twp-converted transcript.
- Error handling: `catcher_create` with a non-NULL diar path that fails to
  load → create fails (the caller explicitly requested diarization). A
  **runtime** diarization error degrades: transcription continues, all
  subsequent text attributes to speaker 0, `catcher_warning` returns a
  message; `catcher_status` stays OK.
- `CatcherClient.swift`: one-line change passing `nil` for
  `diar_model_path` so Tippi keeps compiling; everything else is Phase 3.

### CLI

`catcher transcribe` gains `--diar-model <path>`. With it, output is one line
per segment: `[mm:ss] 說話者N:文字` (N is 1-based for display), Traditional
Chinese; `--json` emits the same JSON as `catcher_segments`. Without
`--diar-model`, behavior is today's plain transcript (now s2twp-converted).

FFI tests: dual-model create, NULL diar path, segments JSON decoding,
degradation path (diar error → warning + speaker 0), lifecycle re-runs.
